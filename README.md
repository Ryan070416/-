# 數位錯題本

## Proposal Report

### 動機與目標
我的微積分一直以來都不太好，考試和作業也常常錯誤百出，所以我希望能夠有更有系統、更方便查看自己曾經錯過的題目或考試壹定會考的經典題型的方法，不然每次都要一一去翻我之前做的筆記很麻煩也消磨我的動機。

### 預期功能
這個程式碼要有使用者介面(tkinter，也就是單機給我自己用)，且至少包含五個功能：

1.歸檔，讓我能夠依照單元分配資料夾來存放題目

2.上傳圖片的功能，如果原始題目是紙本，最好能夠直接拍照放上去，這樣方便很多。且有時候解答也會需要用照片形式儲存。

3.能夠讓使用者(我)自行撰寫解答的區域，可以圖文並茂幫助理解。

4.要有儲存的功能，不然如果每次我用完關掉app它就不見的話就完全失去錯題本的意義了。

5.能夠讓我在每個題目上標注本題難度(用小於2^32的數字)，而同個檔案夾裡面可以利用sorting將難度係數數由大到小/由小到大排序

6.(Optional)如果複習當下覺得解答寫的有點看不懂可以引入gemini AI現場解題，但這個功能我不確定能不能用手上的開發工具做出來，先保留。

### 使用技術
使用文字編輯器sublime，且引進python的函式庫tkinter來完成使用者介面。
關於app關掉要如何儲存的問題，我會想辦法用上學期學的file input功能來完成。
我感覺自己一定會在某些環節卡住，所以會請Gemini來幫忙

### 遇到的困難
<!-- 遇到什麼問題、如何解決或打算如何解決 -->
一開始讓我卡關最久的就是上傳圖片的功能。不知道為何只要選擇上傳圖片，系統就馬上閃退。後來我問AI之後才知道這是平台相容性的問題---當程式呼叫 filedialog.askopenfilename 時，它會去呼叫 macOS 系統原生的「選擇檔案視窗」。這時候，Tkinter 試圖把我們設定的副檔名過濾器 filetypes=[("Image Files", "*.png;*.jpg;*.jpeg;*.bmp")] 丟給 macOS。在 Windows 系統中，用分號隔開副檔名是標準寫法；但在 macOS 中，底層的 Apple API無法解析這個分號字串，導致它在建立副檔名白名單的陣列時，產生了一個空值。而解決的方法是把 filetypes 的寫法，改成 macOS 和 Windows 都能看懂的Tuple格式。這是我原先完全沒想過的問題，但還好有AI幫我上了一課，不然不知道我自己偵錯的話還要debug多久。

目前程式碼沒遇到bug，但我也還沒用極端值(ex:打一個過大的難度數值)來確認程式的try-exception是否正確運行。而且感覺目前的功能還不夠完善，還沒有讓我真正開始使用這個app的動機。
